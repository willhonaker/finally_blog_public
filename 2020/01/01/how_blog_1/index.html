How I Made This Blog - Part 1

An adventure in procrastination.

Intro

When it comes to personal projects, I can be a terrible procrastinator. There’s just too many things to do and too many ways to do them - especially for creating a personal blog.

I’ve made false starts using all of the following and more:

  • Blogger
  • Wordpress
  • Squarespace
  • Jekyll
  • Gatsby
  • Pelican
  • A tumblr page
  • A twitter feed
  • From-scratch static HTML hosted on AWS
  • Enigmatic pictures on Instagram

Since you’re reading this now, you might have inferred that I eventually settled on something - that something is Hugo on Github Pages.

I’m not a Go programmer, so why did Hugo tickle my fancy?

Why I went with Hugo for site building

I don’t know anything about Go, except that it was invented at Google and cleverly named by removing the last four characters from the word google. However, I do like fast websites. Nothing is worse that getting into a good working groove only to have a slow-loading web page throw you off track by giving you five seconds to start browsing the social media vice of your choice.

So I googled: “world’s fastest static site generator”. Lo and behold, Hugo was the first result.

Thanks, Google.

Why I went with Github for hosting

It’s free and easy to use. I also like how they make you feel productive by graphing your commits on a grid over time.

I wasn't very productive in December.

I do make use of a Netlify app to publish the blog instead of publishing directly from the source repo - more on that in the next entry.

Others I did not go with

Pelican

I had the following thought: I use Python for other things already, so why not use the Python static site generator Pelican? I dabbled in it for a few nights, but gave up since I couldn’t find any themes I liked and some of the features that I thought I wanted at the time (like Staticman comments) seemed to take a bit of work to implement. Also, it did not seem to be as fast as other static site generators.

Gatsby

Gatsby looks cool - it’s fast, it’s purple, it has Fortune 500 company logos on the homepage, and you can put React on it. Unfortunately I kept getting sucked into all the tutorials on how to build a site without actually building a site. Next.

Slick.

Jekyll

At one point I wanted to learn Ruby, so Jekyll seemed to fit the bill. Looks simple, fast and easy - and since it’s by the makers of Github hosting is probably a snap. I don’t actually recall why I didn’t go this way - but I suspect it was because I started trying to learn Ruby instead of building a website, only to drop that to pursue other endeavors.

In the next post, I’ll outline step by step how I built this website - follow along to make your own!
